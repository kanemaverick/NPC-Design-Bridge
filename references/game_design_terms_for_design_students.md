# 游戏设计术语速查 — 面向设计学生

本文档为设计学生提供最常见的游戏设计术语和它们对应的设计课语言翻译。

---

## 核心术语

| 游戏设计术语 | 设计课对应概念 | 一句话解释 |
|------------|-------------|----------|
| Core Loop (核心循环) | 用户反复执行的核心操作流程 | 玩家每 2-5 分钟反复做的事 |
| Gameplay Mechanic (玩法机制) | 交互规则 | 游戏里"怎么玩"的规则 |
| Player Experience Goal (玩家体验目标) | 用户情感目标 / Mood | 你希望玩家感受到什么 |
| NPC (Non-Player Character) | 非用户角色 / 服务提供者 | 游戏里不是玩家控制的角色 |
| Game Tone (游戏气质) | 品牌调性 / Visual Mood | 游戏的整体氛围和感觉 |
| Feedback (反馈) | 交互响应 | 玩家做了某事之后游戏给的回应 |
| Progression (进度系统) | 用户成长路径 | 玩家在游戏里怎么变强/变丰富 |
| Balance (平衡性) | 系统均衡 | 游戏难度和奖励是否合理 |
| FTUE (First Time User Experience) | 首次用户体验 / Onboarding | 玩家进游戏前 15 分钟的经历 |
| Sandbox (沙盒) | 开放式探索空间 | 玩家可以自由发挥，不按固定路线 |
| Difficulty Curve (难度曲线) | 学习曲线 | 游戏从简单到难的节奏 |
| Risk-Reward (风险-奖励) | 投入产出比 | 玩家冒多大风险，获得多大回报 |
| Emergent Gameplay (涌现式玩法) | 用户自创用法 | 玩家用游戏规则创造出设计者没预料到的玩法 |

---

## NPC 特定术语

| 术语 | 解释 | 示例 |
|-----|------|------|
| 功能定位 (Function Anchor) | NPC 在核心循环中承担的角色 | 引导者、阻力、奖励、信息源 |
| 接触点 (Touchpoint) | 玩家何时何地遇见 NPC | 游戏开始、任务失败后、路过某地 |
| 行为规则 (Behavior Rules) | NPC 的"当...则..."响应逻辑 | 当玩家失败 → NPC 安慰 |
| 删除测试 (Remove-Test) | 检验 NPC 必要性的方法 | 删除 NPC → 核心循环损失什么？ |
| 节奏调节 (Pace Tuning) | NPC 影响玩家操作速度 | 老奶奶让玩家慢下来 |
| 情感锚点 (Emotional Anchor) | NPC 提供的情感归属 | 让玩家感到安心、被关心 |

## 游戏类型速查

| 游戏类型 | 核心体验 | 典型 NPC 功能 |
|---------|---------|-------------|
| 治愈经营 (Cozy Sim) | 放松、创造、节奏慢 | 情感锚点、节奏调节器 |
| 角色扮演 (RPG) | 成长、探索、故事 | 任务分发者、信息源 |
| 动作冒险 (Action-Adventure) | 挑战、探索、成就 | 引导者、奖励给予者 |
| 恐怖/悬疑 (Horror) | 恐惧、紧张、未知 | 信息碎片分发者、安全区锚点 |
| 模拟/沙盒 (Sim/Sandbox) | 自由、创造、实验 | 目标锚点、资源交换者 |
| 解谜 (Puzzle) | 思考、发现、满足 | 非语言引导者、线索碎片 |
| 社交/互动叙事 | 关系、选择、情感 | 关系推进者、价值观载体 |

---

> 注意：以上术语翻译基于设计课与游戏设计的常见映射关系。在不同游戏类型和设计语境中，具体含义可能会有差异。`[Assumption: 基于游戏设计教学通用实践；可根据目标游戏类型调整]`
