# 作品集自检清单模板

## 使用说明

在完成 NPC Design Bible 后，用此清单逐项检查文档的完整度和质量。每项 0-10 分。

---

## 作品集自检清单 — [NPC 名称]

### 评估时间
[日期]

### 9 项评分

| # | 指标 | 说明 | 分数 (0-10) | 改进建议 |
|---|-----|------|-----------|---------|
| 1 | 体验目标清晰度 | 体验目标是否能用一句话概括？非设计背景的人能否看懂？ | /10 | |
| 2 | NPC 识别度 | NPC 是否有让人记住的特征？外观/说话方式/关系隐喻是否统一？ | /10 | |
| 3 | 核心循环关联 | NPC 是否和游戏的核心循环建立了明确联系？ | /10 | |
| 4 | 玩法功能必要性 | 删除 NPC 后核心循环是否受影响？NPC 功能是否不可替代？ | /10 | |
| 5 | 行为规则可实现性 | 行为规则是否能落地为实际的游戏互动？触发条件是否具体？ | /10 | |
| 6 | 对话功能性 | 对话是否承担玩法功能（而不仅仅是展示性格）？ | /10 | |
| 7 | 玩家接触点具体性 | 接触时刻是否具体？交互内容是否明确？ | /10 | |
| 8 | 作品集表达力 | 文档是否可以被评审/他人独立阅读和理解？排版是否清晰？ | /10 | |
| 9 | 设计决策记录完整性 | 是否记录了关键的设计决策和迭代理由？ | /10 | |
| | **总分** | | **/90** | |

### 评分参照

| 分数段 | 评价 | 说明 |
|-------|------|------|
| 80-90 | 优秀 | 可以直接放入作品集 |
| 60-79 | 良好 | 可以作为作品集素材，但建议补强 |
| 40-59 | 合格 | 核心概念成立，但执行不够深入 |
| 20-39 | 待提升 | 功能定位或玩法关联需要重做 |
| 0-19 | 需重做 | 缺少核心内容 |

### 快速自检（是/否）

- [ ] 体验目标能用一句话说清楚？
- [ ] NPC 有让人记住的特征？
- [ ] 能回答"删掉 NPC 后核心循环缺什么"？
- [ ] 行为规则有具体的触发条件？
- [ ] 对话不只是展示性格，而是服务玩法？
- [ ] 文档可以被别人读懂并继续开发？
- [ ] 留下了设计决策记录？
