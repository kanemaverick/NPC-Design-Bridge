# Step 09: 作品集自检 (Portfolio Review)

## 目标

通过三镜头评审、反套路检测、微型模拟游玩、玩家滥用测试和 9 项评分，完成 NPC Design Bible 的作品集级质量审查。

## 执行流程

### 1. 三镜头评审

从三个视角审查 NPC：
| 镜头 | 检查问题 | 评价 |
|-----|---------|------|
| 玩家镜头 | 我为什么会在意这个 NPC？ | |
| 玩法镜头 | 它改变了我的什么行为？ | |
| 作品集镜头 | 评审能否看懂它的设计价值？ | |

每个镜头给出 1-2 句评价和建议。

### 2. 反套路检测

检查 NPC 设定是否落入常见套路。如果是，给出反差建议：

> [如果检测到常见套路]
> 这个设定目前比较接近常见 NPC 类型。考虑加一个反差变量：
> A. 功能反差（温柔但制造阻力）
> B. 节奏反差（沉默但承担引导）
> C. 奖励反差（不卖东西但改变资源价值）
> D. 关系反差（看似导师其实需要玩家照顾）
> E. 我觉得不需要，保持原样

### 3. 微型模拟游玩

模拟三类玩家体验：

```markdown
### Micro Playtest

| 玩家类型 | 体验评价 | 改进建议 |
|---------|---------|---------|
| Casual Player (休闲) | [评价] | [建议] |
| Efficient Player (效率) | [评价] | [建议] |
| Explorer (探索) | [评价] | [建议] |
```

### 4. 玩家滥用测试

预测玩家可能如何滥用/绕过这个 NPC：

```markdown
### Player Abuse Test

| 风险行为 | 风险描述 | 修正建议 |
|---------|---------|---------|
| [行为 1] | [描述] | [建议] |
| [行为 2] | [描述] | [建议] |
```

### 5. 9 项评分

```markdown
### Evaluation Rubric

| 指标 | 分数 (0-10) | 改进建议 |
|-----|-----------|---------|
| 体验目标清晰度 | /10 | |
| NPC 识别度 | /10 | |
| 核心循环关联 | /10 | |
| 玩法功能必要性 | /10 | |
| 行为规则可实现性 | /10 | |
| 对话功能性 | /10 | |
| 玩家接触点具体性 | /10 | |
| 作品集表达力 | /10 | |
| 设计决策记录完整性 | /10 |
| **总分** | **/90** | |
```

### 6. 作品集导演模式

```markdown
### 作品集展示建议

- **最值得放大展示**：[功能声明 + 删除测试]
- **面试 30 秒话术**：[自动生成]
- **建议补充素材**：[流程图/关系图/玩家旅程图]
```

## 输出格式

完整的作品集自检报告，包含所有 6 个模块。

## 通过标准

- 6 个模块全部执行完成
- 总分已经给出
- 用户收到了作品集展示建议

## DONE 提示

> 🎉 NPC Design Bible 完成！
>
> 你的 NPC 从模糊的灵感变成了可评审、可展示的设计文档。这份文档可以：
> - 放入作品集作为游戏设计能力展示
> - 作为团队开发的 NPC 设计参考
> - 继续迭代，深化设计细节
