# 评估标准 / Evaluation Rubric

本文档定义 NPC Design Bridge Skill 输出质量的评估标准和验证方法。

---

## 自动化验证项（CI 可检查）

### 结构完整性

| 检查项 | 规则 | 方法 |
|-------|------|------|
| SKILL.md YAML frontmatter | 必须包含 `name` 和 `description` | YAML 解析 |
| Prompt 文件存在性 | `prompts/` 下必须有 10 个文件 | 文件计数 |
| NPC Design Bible 必填字段 | 输出必须包含 13 个章节标题 | Regex 匹配标题 |
| 示例目录完整性 | `examples/tea_grandma_cozy_sim/` 必须有 3 个文件 | 文件计数 |

## NPC Design Bible 评分细则（0-10 分制）

### 1. 体验目标清晰度 (0-10)

| 分数 | 标准 |
|-----|------|
| 0-3 | 无体验目标，或体验目标无法被理解 |
| 4-6 | 有体验目标，但描述泛泛（"有趣""好玩"） |
| 7-8 | 体验目标具体可感（"慢、被倾听"） |
| 9-10 | 体验目标具体 + 与游戏类型一致 + 非设计背景也能理解 |

### 2. NPC 识别度 (0-10)

| 分数 | 标准 |
|-----|------|
| 0-3 | NPC 无辨识度，和其他 NPC 无异 |
| 4-6 | 有 1-2 个辨识特征 |
| 7-8 | 外观+说话方式+关系隐喻形成统一的角色印象 |
| 9-10 | 三要素高度统一 + 有让人印象深刻的记忆点 |

### 3. 核心循环关联 (0-10)

| 分数 | 标准 |
|-----|------|
| 0-3 | NPC 与核心循环无关联 |
| 4-6 | 有核心循环描述，但 NPC 不在循环中 |
| 7-8 | NPC 明确嵌入核心循环的某个环节 |
| 9-10 | NPC 深度嵌入循环 + 对循环节奏有调节作用 |

### 4. 玩法功能必要性 (0-10)

| 分数 | 标准 |
|-----|------|
| 0-3 | 删除 NPC 后游戏完全不受影响 |
| 4-6 | 删除 NPC 后有轻微影响，但不致命 |
| 7-8 | 删除 NPC 后在 2+ 个维度有明显损失 |
| 9-10 | 删除 NPC 后在 4 个维度都有实质损失，NPC 功能不可替代 |

### 5. 行为规则可实现性 (0-10)

| 分数 | 标准 |
|-----|------|
| 0-3 | 无行为规则，或规则无法落地 |
| 4-6 | 有行为规则，但触发条件模糊 |
| 7-8 | 每条规则有具体触发条件 + NPC 反应 + 玩家影响 |
| 9-10 | 规则具体 + 涵盖正常和异常玩家行为 + 有边界条件 |

### 6. 对话功能性 (0-10)

| 分数 | 标准 |
|-----|------|
| 0-3 | 无对话，或对话纯文学表达 |
| 4-6 | 有对话，但仅展示性格 |
| 7-8 | 对话承担了玩法功能（引导/信息/奖励） |
| 9-10 | 对话同时履行玩法功能 + 展示性格 + 推动节奏 |

### 7. 玩家接触点具体性 (0-10)

| 分数 | 标准 |
|-----|------|
| 0-3 | 无接触点，或接触点过于泛泛 |
| 4-6 | 有 1-2 个接触点，但时刻不够具体 |
| 7-8 | 有 3 个具体接触点 + 每个有交互内容 |
| 9-10 | 接触点覆盖游戏的初期/中期/后期 + 每次交互有差异化产出 |

### 8. 作品集表达力 (0-10)

| 分数 | 标准 |
|-----|------|
| 0-3 | 文档无法被他人独立阅读 |
| 4-6 | 文档可读，但结构混乱或缺少关键章节 |
| 7-8 | 文档结构清晰、格式统一、可被他人阅读 |
| 9-10 | 文档有专业设计文档的质感 + 可读性高 + 有视觉层次 |

### 9. 设计决策记录完整性 (0-10)

| 分数 | 标准 |
|-----|------|
| 0-3 | 无决策记录 |
| 4-6 | 有决策记录，但缺少"为什么" |
| 7-8 | 记录了关键决策和变更原因 |
| 9-10 | 完整的决策-原因-影响链，可作为后续迭代的参考 |

## 总分与等级

| 总分范围 | 等级 | 建议 |
|---------|------|------|
| 80-90 | A — 优秀 | 可直接放入作品集 |
| 60-79 | B — 良好 | 可作为作品集素材，建议补强弱项 |
| 40-59 | C — 合格 | 核心概念成立，但执行不够深入 |
| 20-39 | D — 待提升 | 功能定位或玩法关联需要重做 |
| 0-19 | F — 需重做 | 缺少核心内容 |
