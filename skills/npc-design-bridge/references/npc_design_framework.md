# NPC 设计框架 / NPC Design Framework

本文档为 NPC Design Bridge Skill 提供 NPC 设计的基础理论框架，供 Skill 在执行过程中参考。

---

## 核心概念

### 什么是"设计语境桥接"（Design Bridge）

设计学生熟悉的工具和语言与游戏设计语言之间存在鸿沟。Design Bridge 在两者之间建立映射：

| 设计课概念 | 游戏设计概念 | 桥接方式 |
|-----------|------------|---------|
| 情绪/氛围 (Mood) | 玩家体验目标 (Player Experience Goal) | "你希望玩家感受到什么情绪？" → "玩家体验目标是什么？" |
| 用户画像 (Persona) | 角色概念卡 (Character Concept Card) | 外观/行为/场景 → 外观/说话方式/关系隐喻 |
| 接触点 (Touchpoint) | 玩家触发时机 (Player Trigger) | 用户接触点 → 玩家何时何地遇到 NPC |
| 服务流程 (Service Flow) | 核心循环 (Core Loop) | 用户反复做什么 → 玩家反复做什么 |
| 角色关系 (User Relationship) | NPC 功能定位 (NPC Function) | 品牌给用户什么？ → NPC 给玩家什么？ |

### 什么是"玩法锚定"（Gameplay Anchor）

玩法锚定是将 NPC 的设计价值锚定在游戏机制上的方法。核心检验标准：

> **删除测试**：如果从游戏中删除这个 NPC，核心循环的玩法、情绪、信息、节奏分别损失什么？

一个通过了删除测试的 NPC 必然与游戏机制深度绑定。

## NPC 功能罗盘

NPC 在游戏中的功能可以映射到以下罗盘：

```
             情感锚点 (Emotional Anchor)
                  ↑
                  |
 阻力 (Resistance) ← → 奖励 (Reward)
                  |
                  ↓
             信息源 (Information Source)
```

### 六种细分功能类型

| 功能类型 | 说明 | 典型表现 | 适合的游戏类型 |
|---------|------|---------|-------------|
| 节奏调节器 | 控制玩家的操作节奏 | 让玩家慢下来/催促玩家 | 治愈经营、动作游戏 |
| 风险制造者 | 制造决策压力和信息不对称 | 提供有代价的选择 | RPG、策略游戏 |
| 目标提示者 | 给玩家方向和目标 | 告诉玩家下一步去哪 | 开放世界、解谜游戏 |
| 世界观入口 | 传递世界观和叙事 | 讲述故事、展示世界 | 叙事游戏、开放世界 |
| 关系推进者 | 与玩家建立情感连接 | 记忆、感情变化 | 社交模拟、RPG |
| 资源交换者 | 提供物品/信息交换 | 买卖、交易 | 所有类型 |

## NPC 设计原则

1. **功能优先**：先确定 NPC 在玩法中的功能，再设计外观和性格
2. **不可替代性**：每个 NPC 应该承担一个不可被其他机制替代的功能
3. **玩家影响可见**：NPC 的存在应该改变玩家的行为或体验
4. **边界清晰**：NPC 应该有自己的规则和限制，不是玩家的万能工具
5. **经济性**：用最少的特征传递最强烈的印象（少即是多）

## 常见设计陷阱

| 陷阱 | 表现 | 修正方向 |
|-----|------|---------|
| 纯人设陷阱 | NPC 只有外观和背景故事，无玩法功能 | 运行删除测试 |
| 万能 NPC 陷阱 | NPC 什么功能都承担，变成"瑞士军刀" | 限定 1-2 个核心功能 |
| 任务布告板陷阱 | NPC 只是分发任务的工具人 | 赋予 NPC 自己的动机和限制 |
| 信息倾倒陷阱 | NPC 的对话是长段世界设定介绍 | 把信息分散到多个接触点和互动中 |
| 情绪绑架陷阱 | NPC 过度可爱/可怜，让玩家出于情感而非玩法关注它 | 确保玩法功能仍然成立 |
