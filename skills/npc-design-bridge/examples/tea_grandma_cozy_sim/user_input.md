# 用户输入示例：温柔老奶奶

## 用户的初始输入

> 我想做一个温柔的老奶奶 NPC，喜欢喝茶，慢、耐心，希望她出现在治愈经营类游戏里，但我不知道她在玩法里有什么用。

## Skill 识别的关键信息

| 维度 | 提取内容 |
|-----|---------|
| NPC 类型 | 老奶奶，温柔，喜欢喝茶 |
| 性格关键词 | 慢、耐心 |
| 游戏类型 | 治愈经营类（用户明确提出） |
| 用户痛点 | 不知道 NPC 在玩法中的作用 |
| 适合的反差方向 | 功能反差（温柔但制造阻力）或 节奏调节（慢 = 帮玩家慢下来） |

## Skill 介入策略

1. 用户已提供了游戏类型（治愈经营），可以跳过 Step 1 的游戏类型提问，直接进入体验目标确认
2. 用户已有一定角色概念（老奶奶、喝茶、慢、温柔），可以在 Step 2 快速确认
3. 用户的核心痛点是"不知道玩法作用"，这是 Step 4-5 需要重点攻克的地方
4. 建议引导方向：老奶奶的"慢"可以转译为"节奏调节器"功能
