# 做中学体验评估 / Learning Experience Rubric

本文档从"做中学"体验质量的角度评估 NPC Design Bridge Skill 的执行效果。
每个维度 0-10 分，共 50 分。

---

## 1. 做中学感 (0-10)

**评估重点**: Skill 是让用户填表，还是让用户在设计中学习？

| 分数 | 标准 |
|-----|------|
| 0-3 | 只是让用户填表，或直接替用户生成 |
| 4-6 | 有少量解释，但和用户当前设计脱节 |
| 7-8 | 每步都有小设计动作和原理解释，用户能感知到因果关系 |
| 9-10 | 用户能清楚感到自己在通过设计动作学习专业方法，并能将方法迁移到自己的设计实践中 |

---

## 2. 设计根源可信度 (0-10)

**评估重点**: Skill 的建议是像主观判断还是有据可查？

| 分数 | 标准 |
|-----|------|
| 0-3 | 建议像主观判断——"我建议这样做因为这样更好" |
| 4-6 | 有理论或案例，但没有说明与当前设计的具体关系 |
| 7-8 | 建议能追溯到术语、模板、案例或设计原则，且说明与当前 NPC 的关系 |
| 9-10 | 每个关键建议都有清晰依据链：用户输入 → 设计原则 → 模板/案例 → 当前 NPC 的修改建议 |

---

## 3. 模板协作透明度 (0-10)

**评估重点**: Skill 是用模板帮用户，还是暗中套用模板？

| 分数 | 标准 |
|-----|------|
| 0-3 | 暗中套模板，用户不知道用了模板 |
| 4-6 | 提到模板但没有让用户选择和确认 |
| 7-8 | 推荐候选模板并说明适配理由和风险，让用户确认 |
| 9-10 | 用户确认模板选择 + 模板套用后有反套路优化 + 模板选择过程被记录 |

---

## 4. 案例比对质量 (0-10)

**评估重点**: 案例是比较还是学习工具？

| 分数 | 标准 |
|-----|------|
| 0-3 | 只是列出相似角色（"你的 NPC 像 XXX"）|
| 4-6 | 有案例说明，但没有转化为当前 NPC 的设计建议 |
| 7-8 | 能比较当前 NPC 和案例的相似点、差异点、风险和可借鉴模式 |
| 9-10 | 能从案例中提炼设计模式，给出"可借鉴点"和"不应照抄点"，转化为具体修改建议 |

---

## 5. 决策链完整度 (0-10)

**评估重点**: 最终 Bible 是否包含完整的推理链？

| 分数 | 标准 |
|-----|------|
| 0-3 | 没有记录设计决策 |
| 4-6 | 有记录，但缺少依据 |
| 7-8 | 记录了决策、依据和影响 |
| 9-10 | 能形成完整 Reasoning Trail——每一步的决策、依据、放弃方案和玩家影响都可追溯，可用于作品集展示 |

---

## 总分与等级

| 总分范围 | 等级 | 含义 |
|---------|------|------|
| 40-50 | A | 优秀的学习设计体验 |
| 30-39 | B | 基本实现了"做中学"目标 |
| 20-29 | C | 有学习要素但体验不足 |
| 0-19 | D | 未能实现做中学目标 |
