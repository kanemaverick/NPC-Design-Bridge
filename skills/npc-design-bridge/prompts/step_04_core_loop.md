# Step 04: 核心循环 (Core Loop)

## 目标

用填空引导用户描述游戏的核心循环，明确 NPC 所处的玩家行为节奏。

## 执行流程

### 问题 1：填空引导

> 在你的游戏里，玩家反复做什么事？不用写游戏设计术语，填空就好：
>
> 玩家反复做 _______
> 得到 _______
> 感受 _______
>
> 写个大概就行，后面可以改。

### 问题 2：参考游戏

> 如果不确定核心循环，告诉我这个游戏最像哪个已有的游戏？
>
> 比如："像《动物森友会》"、"像《星露谷物语》"、"像《传说之下》"
>
> 我会根据你提到的参考游戏，帮你推导一个循环。你只需要确认或修改就好。

> 🎨 设计引导：核心循环就是玩家 2-5 分钟内的行为节奏。比如"泡茶 → 招待客人 → 得到配方 → 解锁故事"就是一个循环。先填空，我会帮你整理成设计语言。

### 问题 3：循环确认

根据用户填空，Skill 生成完整的核心循环描述：

> 我帮你整理了一下：玩家反复 [动作]，得到 [奖励]，感受到 [体验]。这个循环里，每个周期大约 [X] 分钟。对吗？

## 输出格式

```markdown
### 核心循环 (Core Gameplay Loop)

**玩家行为循环**:
> 玩家反复 [做 X] → 得到 [Y] → 感受 [Z]

**循环节奏**: 每个周期约 [X] 分钟

| 设计语言 | 游戏设计语言 |
|---------|------------|
| 反复做的事 | 核心循环 (Core Loop) |
| 行为节奏 | 循环周期 (Loop Duration) |
```

## 通过标准

- 用户至少填了 1 个空（玩家反复做什么）
- 有了一句核心循环描述（AI 辅助生成或用户自述）

## 不通过处理

- 用户完全没想法 → 提供类型模板：
  > 那我们先定游戏类型。你觉得更像哪种？
  > A. 经营类：收集资源 → 建造/制作 → 吸引客人 → 获得新资源
  > B. 探索类：进入新区域 → 发现线索 → 解开谜题 → 解锁下一区域
  > C. 社交类：遇见角色 → 对话/帮助 → 提升关系 → 解锁新故事
  > D. 养成类：训练/学习 → 升级 → 解锁能力 → 挑战更难内容
  > E. 不确定，我再想想
